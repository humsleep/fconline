# 혁신 기능 회의 결정문 (2026-07, 4라운드 합의)

5개 페르소나(PM/그로스·UX/비주얼·성능·크리에이티브·영게이머) 다라운드 토론 종합.

## 방향

"조회·나열형" 경쟁사와 결별 → **독점 데이터(슛좌표·spRating·패스7종·랭커갭)를
룰베이스 '심판(Verdict) 엔진'으로 판정 → 공유 가능한 이미지 카드로 출력.**
AI 호출·비용·지연 0 (전부 룰 + 캐시).

## 롤아웃 순서 (의존성 확정)

`Verdict 엔진 → 카드 엔진 → 스트리밍 리빌 로딩 → 랭커 프리컴퓨트 → VS 판독기 → 3초 위계·성능예산 계측`
※ VS 판독기는 랭커 프리컴퓨트(크론) 배포 완료가 착수 전제.

## 속도

- SSE 2단계 하이드레이션: 메타로 스켈레톤 즉시 → detail 도착 순서대로 in-place 교체(CLS 0)
- 기본 5경기 + "더 보기" 30경기 opt-in
- 유저 경로에서 넥슨 동기 호출 제거 — 랭커/VS는 프리컴퓨트+캐시 뒤로
- VS 조합 (A,B,pos)은 순수함수 → 1회 계산 후 무기한 캐시
- 성능 예산: 첫 카드 ≤1.0s / 인터랙션 ≤1.5s / 캐시히트 ≤100ms

## 시각화 (텍스트/표 → 비주얼)

| 지금 | 전환 |
|---|---|
| 매치 결과 나열 | 전광판 카드(슛맵 + verdict 도장 + 평점 배지) |
| spRating 숫자 | 티어+밈등급 도장 스탬프(색+형태 이중 인코딩) |
| 패스 7종 표 | 미니 게이지, 낮은 항목만 라임 강조 |
| 랭커 갭 수치 | tug-of-war 3바(내 vs 랭커, 넘으면 발광) |
| 슛 좌표 | 골대 배경 슛맵 히트존 + 골/노골 팝인 |
| VS 비교표 | 좌우 슬램 카드 + 포지션 라벨 스왑 3바 |
| 로딩 스피너 | 도트매트릭스 카운터 + 카드 순차 팝인 |
| 공유 | 9:16 OG PNG(헤드라인/히어로/스탬프+배지3/QR) |

승패·우열은 반드시 **색 + 형태 이중 인코딩**(색맹 대응).

## 청소년·대학생 UX 원칙

1. 3초 위계: 화면당 판정 1 + 히어로 1 + 배지 최대 3, 나머지 접기
2. 홈 첫 스크린 = "오늘의 VS" 히어로 1개
3. 결론부터(티어+한줄판정), 근거는 아래로
4. 최우선 CTA = "이미지 저장" 원탭, 로그인 강요 X
5. 딥링크는 홈이 아니라 "방금 본 그것"으로 착지

## Sprint 3 = VS 판독기

선행(하드 게이트): 랭커 프리컴퓨트 + 캐시 배포 완료.
흐름: 슬램 등장 → 예측 소프트게이트 투표(탈출구 O) → 정답 공개(tug-of-war 3바 + 승자 발광 + 심판 도장) → 적중 뱃지(localStorage) → 9:16 카드 공유.
투표: 진입 스냅샷 % + optimistic 업데이트, 실시간 폴링 금지.
데일리: "오늘의 VS" 스탯 근접쌍 자동 편성(운영 개입 0), localStorage 스트릭.

## 하지 말 것

1. 유저 경로 넥슨 동기 호출 금지 (프리컴퓨트/캐시 뒤로)
2. 실유저(otherUser) 대상 하드 톤 판정 금지 — `subjectType` 3단 래더(self/player/otherUser)로 코어 차단. 저격·굴욕 톤은 player/self만. 친구 VS는 상호동의 구조 갖춰지는 Sprint 4로 유보.

## 미결 (방향 합의, 확정만 대기)

1. 친구 VS 개봉 = Sprint 4 + 상호동의 초대 구조
2. K-factor 게이트 임계값 = 주간 카드생성 baseline 실측 후 결정. 지금은 UTM 관측만.
