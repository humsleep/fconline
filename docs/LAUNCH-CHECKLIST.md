### 라이브 오픈 하드닝 (회의 5R 결과 — 코드 반영 완료)
- ✅ **넥슨 kill-switch**: `/admin`에서 배포 없이 넥슨 팬아웃 즉시 정지/재개 (fail-open — Supabase 순단이 넥슨 호출을 막지 않음)
- ✅ **비로그인 넥슨 팬아웃 IP rate limit**: `/api/squad/from-user`·`/api/live` IP당 분당 40회(관대 — CGNAT 감안), 429+Retry-After
- ✅ **넥슨 장애 정직 안내**: 전적 페이지 429/타임아웃/일시정지 전용 안내(재시도 연타 유도 안 함), 장애를 "경기 없음"으로 위장하지 않음
- ✅ **신뢰성 자가치유**: 랭커 tombstone은 성공한 청크만 기록(넥슨 순단 1회로 하루 열화 방지), 선수 인덱스 로드 실패는 캐시 안 함(자동 재시도)
- ✅ **커뮤니티 작성 간격 제한**: 글 30초·댓글 10초 (RLS + API 이중 방어)
- ✅ **공유 OG 이미지**: 스쿼드/매치 링크에 카드 썸네일 + sitemap에 공개 스쿼드/글 포함
- ⏸ **DEFER(오픈 후)**: 자동 429 백오프(크론 자가치유 역회전 위험), 분산 KV rate limit, match_cache TTL 크론, 개인 기록 축적, 사칭/신고 어뷰징 정교 대응

## 3.5) 🔴 /meta 픽 랭킹 시딩 (오픈 전 필수 — 회의 P0 1순위)

픽 랭킹은 전적 검색 데이터(match_cache)에서 만들어지므로 오픈 첫날은 확정적으로 빈 화면.

> ℹ️ 넥슨 `/fconline/v1/ranker-stats`는 "랭커 목록"을 주는 API가 아니라, 우리가 지정한
> 선수×포지션 조합의 랭커 평균 스탯을 돌려주는 **조회형** API라 시딩을 단독 대체할 수 없다.
> 대신 아래 원클릭 도구가 수동 검색 단계 자체를 없앤다.

오픈 전 시딩 — **`/admin` → "픽 랭킹 시딩"** 한 번이면 끝:

1. 활동 중인 구단주명(공식 랭킹 상위권/지인) **최대 5개 입력 → 시딩 실행**
   - 서버가 닉네임당 최근 공식경기 30판을 수집(match_cache 축적)하고,
     방금 수집한 경기에서 자주 쓰인 선수×포지션 상위 60개 조합의 랭커 스탯 워밍까지 자동 처리
   - 크론 수동 실행(curl) 불필요. 더 채우고 싶으면 다른 닉네임으로 2~3회 반복
2. `/meta` 접속 → 공격/미드/수비/GK 4개 라인이 채워졌는지 확인 (1시간 캐시라 최대 1시간 후 반영)
3. 이후는 자동: 크론이 매일 갱신하고, 검색이 없어도 직전 스냅샷 조합을 재예열(폴백 구현됨)
4. §5 실데이터 검증에서 본인 계정을 검색하는 것도 최소 시딩을 겸함

변동 배지(▲▼)는 이틀치 스냅샷이 필요하므로 시딩 다음날 크론 이후부터 표시된다.

시딩이 어려우면 오픈 시 /meta를 내비에서 임시 숨김(홈 카드 + 헤더 + 모바일 탭 3곳).
